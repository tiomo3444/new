
# V2.1 修正版執行步驟

## 為什麼上一版發生錯誤？

你的 Supabase 曾經執行過較早版本的 SQL，因此已經存在一張舊的 `deliveries` 表。

舊表沒有 `customer_id` 欄位，而新版 SQL 使用 `create table if not exists`，因此沒有覆蓋舊表，最後建立索引時便出現：

`column "customer_id" does not exist`

V2.1 會先刪除尚未正式使用的舊物流資料表，再完整重建。

## 執行步驟

1. 回到 Supabase SQL Editor。
2. 把目前編輯器中的 SQL 全部刪除。
3. 打開 `furniture_delivery_v2_1_reset.sql`。
4. 全選並複製全部內容。
5. 貼進 Supabase SQL Editor。
6. 按右上角 `Run`。
7. 出現 `Potential issue detected` 時，按 `Run query`。
8. 等待執行完成。

成功時應顯示：

`Success. No rows returned`
